# Mnemo v0.2.0-preview.1

Windows 预览构建（prerelease）。本次版本是一次全面的质量提升，涵盖安全加固、架构治理、PikPak 长视频播放对齐 rclone 以及工程化改进。

## 本次更新内容

### 🔒 安全加固

**渲染进程安全**
- 启用 `contextIsolation` + 禁用 `nodeIntegration`，彻底阻断 XSS→RCE 攻击链
- 用 `contextBridge` 重写 preload，移除 `window.Electron` 与 `TvBoxInvoke` 全局暴露
- 所有 IPC 通信改为白名单 API，不再暴露完整 Electron 模块

**凭据与密钥**
- aria2 RPC secret 改为随机生成并持久化到用户数据目录，含旧硬编码密钥自动迁移
- Dropbox OAuth 始终启用 PKCE（即使使用内置 client secret）

**IPC 安全**
- `WebShutDown` 增加 `confirm: true` 校验，防止误触发系统关机；Linux 改用 `systemctl poweroff`
- `WebSetCookies` 域名计算修正；cookie / storage API 输入白名单校验
- `shell.openPath` 迁移至主进程并校验路径（仅允许打开文件，拒绝目录）

**XSS 防护**
- 批量重命名预览全量 `escapeHtml`，修复 `v-html` 渲染未转义文件名的 XSS 漏洞（高亮标记保留，文件名内容转义）

### 🛡️ 稳定性与健壮性

**退出路径修复**
- 修复 aria2c 引擎在退出时孤立：`will-quit` 使用 `event.preventDefault()` + 异步清理 + `isQuitting` 防重入
- exit / relaunch / 关机路径均 `await` 引擎关闭后再退出

**引擎管理**
- `Engine.start()` 增加 spawn `error` 事件处理
- 陈旧 PID 清理：启动时检测并杀死残留 aria2c 进程
- `ensureEngineReady` 真正等待引擎 RPC 就绪

**上传 Worker**
- 崩溃重试上限 5 次，超限发送错误消息（防止无限重启循环）

**Provider 健壮性**
- Dropbox：令牌刷新单飞（防并发竞态）、搜索分页（10 页上限）
- OneDrive：上传分片 3 次重试 + 退避、令牌刷新单飞
- Google Drive：上传分片 3 次重试 + 退避
- S3：大文件分片复制（>5GB 使用 `UploadPartCopy`）
- WebDAV：流式上传（替代整文件读入内存）、进度回调改为增量（修复 sync 二次累加）

### 🎬 PikPak 长视频对齐 rclone

本次重点修复长视频播放可靠性，与 rclone PikPak backend 对齐：

- **链接刷新单飞锁**：过期边界的并发 Range请求合并为一次 API 调用，防止触发限流
- **JSON `link.expire` 回退**：URL 无 `expire` 参数时解析 JSON 字段（RFC3339）作为过期时间，恢复主动无缝刷新
- **转码流 fid 校验**：过滤 fid 不匹配的转码链接，防止选择失效转码流
- **文件详情重试**：无可用链接时重试 2 次（3s 延迟），修复刚离线下载的长视频首次打开失败

（MPEG-TS 转码解码、401/403 响应式刷新、Origin fid 校验等已在此前版本对齐。）

### 🏗️ 架构治理

**死代码清除**
- `alihttp.ts` 精简（~490 行 → 147 行），移除阿里云令牌管理 / 错误码 / 重试逻辑
- 移除 `isAliyunUser()` 等阿里云专用函数与残留 `console.log`

**标签系统重构**
- 独立数据模型：`FileTagDef`（标签定义）+ `FileTagLink`（文件-标签关联），与快捷方式 / 收藏分离
- 标签 CRUD：创建、重命名、删除（级联删除关联）
- 版本化存储 + 一次性迁移、localStorage 缓存、孤立链接清理
- 标签 ID 使用 `crypto.randomUUID()` 防碰撞
- 所有网盘支持（localStorage 存储，Provider 无关）

**内存优化**
- `treestore` 增加 `clearTreeCache()`，账户切换时清理目录树缓存
- `PageMain` 的 `timeEvent` 可停止；`PageImage` 的 Viewer.js 实例在 unmount 时销毁

### 🔧 工程化

- `tsconfig.node.json` 启用 `strict: true`，修复全部 strict 错误
- 添加 ESLint（flat config：`@typescript-eslint` + `eslint-plugin-vue` + `eslint-plugin-security`）+ lint 脚本 + CI 集成
- 新增 PR / push CI 工作流（typecheck + test + lint）
- 依赖整理：运行时依赖归位、`@types/node` 升级 22、移除 `dom-to-image`
- `electron-builder` 移除无条件 `notarize`；`vite` `strictPort: true`、console drop 精细化
- `.gitattributes` 配置 Git LFS（未来二进制走 LFS）
- `index.html` 移除阿里巴巴残留；`launch.ts` 移除阿里云 header 注入

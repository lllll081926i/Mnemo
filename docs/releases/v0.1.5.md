# Mnemo v0.1.5

**正式版（stable）**。在 `v0.1.2` / `v0.1.3-preview.1` 之上整合 PikPak 长视频播放修复、文件夹双向同步、UI 与下载链路加固，并彻底统一全平台安装包的产品名与图标。

## 下载说明

| 平台 | 架构 | 推荐安装包 |
|------|------|------------|
| Windows | x64 / arm64 | `.exe`（NSIS 安装程序） |
| macOS | Intel (x64) / Apple Silicon (arm64) | `.dmg` 或 `.zip` |
| Linux | x64 / arm64 | `.AppImage` / `.deb` / `.pacman` |

- 产品名：**Mnemo**
- 可执行文件：Windows `Mnemo.exe` · macOS `Mnemo.app` · Linux `mnemo`
- 应用 ID：`com.mnemo.app`
- 版本号：`0.1.5`（正式版，非 preview）

## 本版重点

### 安装包品牌（名称 + 图标）

- 统一使用标准 `build/` 图标资源：`icon.ico` / `icon.icns` / 多尺寸 `icons/`
- Windows / macOS / Linux / NSIS 安装器、快捷方式、卸载显示名全部为 **Mnemo**
- 可执行文件写入产品名、公司名与图标（`signAndEditExecutable`）
- 运行时窗口、托盘与 About 面板通过 `getAppIconPath()` 解析图标，避免回退到 Electron 默认壳

### PikPak 长视频与播放

- 修复长视频「完全无法播放」：转码流为 MPEG-TS 时的识别与播放链路
- 长视频预览加固：对齐 rclone 的链接管理策略，降低链接失效与黑屏概率

### 网盘能力与同步

- 文件夹**双向同步**任务（本地 ↔ 云端）
- 文件类型识别修复
- 本地标签快捷方式体验修复
- 第三方 OAuth 登录相关修复（OneDrive / Dropbox / Google Drive 等）

### 稳定性与体验

- 同步任务 / 系统代理 / 播放器相关多项审查修复
- Aria 下载：RPC secret 与端口一并回传，避免引擎鉴权不一致
- 统一全部弹窗布局风格，交互更一致

### 构建与发布

- GitHub Actions 全平台矩阵：Windows / Linux / macOS × x64 / arm64
- tag 推送后：类型检查 + 测试 → 并行打包 → 上传 Release 与 `SHA256SUMS.txt`
- 正式版构建校验 OneDrive / Dropbox / Google Drive 完整 OAuth 配置

## 已知说明

1. **Office 文档**：不提供应用内预览，请下载后用系统应用打开。
2. **PikPak 验证码**：若反复失败，可使用登录页「重置设备 ID」，并稍候再试。
3. **GDrive 配额**：共享 OAuth 项目可能触发限流；已有退避重试，仍失败可稍后再登。
4. **分享页记录**：仅本机创建过的链接；清除应用数据会丢失本地分享历史。
5. **同步任务**：首次配置请确认本地路径与云端目录权限，避免误覆盖。

## 安装与升级

1. 建议先退出正在运行的 Mnemo；可卸载旧预览版（用户数据目录默认同用 `Mnemo`）。
2. 安装对应平台安装包。
3. 若任务栏 / 快捷方式仍显示旧图标，重启资源管理器或清理 Windows 图标缓存后再查看。
4. 首次启动检查各网盘登录状态。

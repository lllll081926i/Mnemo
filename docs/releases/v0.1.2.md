# Mnemo v0.1.2

**首个多云正式版（stable）**。在 `v0.1.2-preview.1` 预览构建之上，完善 PikPak 登录与播放链路、文件列表体验、分享与标签，并统一全平台打包与产品标识。

## 下载说明

| 平台 | 架构 | 推荐安装包 |
|------|------|------------|
| Windows | x64 / arm64 | `.exe`（NSIS 安装程序） |
| macOS | Intel (x64) / Apple Silicon (arm64) | `.dmg` 或 `.zip` |
| Linux | x64 / arm64 | `.AppImage` / `.deb` / `.pacman` |

- 产品名：**Mnemo**（不再使用 Electron 默认壳标识）
- 应用 ID：`com.mnemo.app`
- 版本号：`0.1.2`（正式版，非 preview）

## 本版重点

### PikPak 登录与 API
- 验证码流程对齐官方协议：令牌换发退避重试、回调解析 `captcha_token`、设备 ID 可重置
- **所有 Drive API 请求附带 `X-Captcha-Token`**（按 action 缓存、链式换发、失效自动重试），修复「登录成功但列表为空 / 验证码无效」
- 登录页增加「设备被限制登录？重置设备 ID」入口，并清理验证码会话分区

### 在线播放（PikPak）
- `medias` 映射多档清晰度；**会员可用原画，非会员最高 720p**
- 原画优先使用 progressive 媒体流，下载始终走原文件直链
- 播放失败自动降档尝试，降低大文件 / 长文件名场景黑屏概率

### 文件列表与网盘操作
- 列表行：日期 / 时间两行居中、行间距与圆角、勾选框与工具条左右对齐
- 缩略图模式：选中框层级修正，缩略图失败时回退文件图标（避免与图标叠层）
- 文件夹不再误用内部视频封面作为图标
- **PikPak 回收站**按官方参数（`parent_id=*`）修复
- 文件夹属性：静默缓慢递归统计子文件，本地缓存 + 签名失效（**改名不重算**，增删移才重算）
- 本地标签：快捷方式水合、标记筛选视图、行内颜色图标；打标 / 清除异常可提示
- 云下载 / 重命名弹窗布局重做；批量重命名规则面板**默认折叠**，工具条左对齐

### 分享
- 创建分享后写入**本地分享记录**
- 分享页：按账号聚合、网盘 / 账号 / 状态筛选、关键词搜索、复制链接与提取码、删除记录
- 当前版本不提供第三方分享导入与云端分享列表同步（以应用内创建记录为准）

### 其它稳定性
- Google Drive：配额 / 429 退避重试，降低共享 OAuth 项目限流导致的登录失败
- Aria 下载：修复 `Info` 冻结后无法写回 GID 导致任务失败
- 设置：写盘防抖、配置损坏备份、标签编辑器重做；页面切换动画修复
- **移除 Office 在线预览**（docx / 表格 / LibreOffice 转 PDF）；Office 类文件请下载后用系统应用打开
- 保留：PDF、文本 / 代码、图片、音视频预览

### 构建与发布
- GitHub Actions **全平台矩阵**：Windows / Linux / macOS × x64 / arm64
- 安装包产品名、快捷方式、卸载显示名、安装器图标统一为 **Mnemo**
- 正式版 tag 触发严格 secrets 校验与 Release 发布

## 已知说明

1. **Office 文档**：不再提供应用内预览，请下载后打开。
2. **PikPak 验证码**：若仍反复失败，可使用登录页「重置设备 ID」，并稍候再试（避免触发设备限制）。
3. **GDrive 配额**：若出现 `Quota exceeded`，多为共享客户端项目的每分钟配额；本版已自动退避重试，仍失败时可稍后再登。
4. **分享页记录**：仅记录在本机创建过的链接；清除应用数据会丢失本地分享历史。
5. **文件夹统计**：超大目录可能显示「已达统计上限」；属性能保护，非错误。

## 安装与升级

1. 卸载旧版预览包（可选；用户数据目录默认同用 `Mnemo`）。
2. 安装对应平台安装包。
3. 首次启动检查网盘登录状态；PikPak 若遇设备限制请重置设备 ID 后重登。

## 质量验证

- 单元 / 集成测试全量通过（约 63 套件 / 297 用例，以 CI 日志为准）。
- 渲染进程与 Electron 主进程 TypeScript 检查通过。
- 正式版由 GitHub Actions 多平台流水线构建并上传到本 Release。

## 与 preview 的关系

- `v0.1.2-preview.1`：预览通道，侧重死代码清理与多云列表统一。
- **`v0.1.2`**：正式通道，在 preview 基础上完成 PikPak 可用性、UI 体验与全平台发布管线。

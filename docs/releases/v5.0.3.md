# BoxPlayer v5.0.3

> 发布主题：139 云盘访问修复、115 云下载目录选择修复、账号配置安全收口。

## 发布摘要

v5.0.3 是基于 v5.0.2 的问题修复版本，重点处理用户反馈的两类网盘问题：

- 修复 139 云盘文件列表、预览和第三方直链访问中的兼容性问题。
- 修复 115 网盘云下载无法自定义选择保存文件夹，尤其是根目录选择失效的问题。

同时，本版本继续收口 BoxPlayer 账号登录相关配置，Supabase publishable key、站点地址等客户端配置改为通过 `.env.local` / GitHub Secrets 生成，避免继续硬编码在 `src/` 源码中。

## 重点修复

- 修复 139 云盘 PersonalCloud 路由地址拼接错误，避免出现重复 `/orchestration/orchestration/` 或错误路径。
- 修复 139 云盘新旧接口返回结构不一致导致的文件列表为空问题，兼容 `items`、`catalogList`、`contentList` 等返回格式。
- 修复 139 云盘请求签名时间格式和请求头兼容性问题，补齐 Web 端所需的 `mcloud-sign`、客户端版本、渠道等字段。
- 修复 139 云盘 PDF、图片、音频、视频和文件属性预览误走阿里云盘接口的问题，第三方网盘会使用原始下载地址或本地代理预览。
- 修复 115 网盘云下载保存到根目录时没有传递 `wp_path_id=0` 的问题。
- 修复从网盘页面打开 115 云下载时，当前目录为根目录却回退到默认离线下载目录的问题。
- 修复云下载选择目录弹窗中 115 根目录显示与提交值不一致的问题。

## 账号配置

- BoxPlayer app 侧的 `SUPABASE_PUBLISHABLE_KEY`、`BOXPLAYER_SUPABASE_URL`、`BOXPLAYER_SITE_URL` 已接入生成式 secrets 流程。
- GitHub Actions Release 构建会从 GitHub Secrets 生成 `src/secrets.generated.ts`。
- `.env.local` 和 `src/secrets.generated.ts` 仍保持 gitignore，不会提交真实配置。
- app 侧订阅状态刷新遇到 `invalid_auth_token` 时会主动刷新 Supabase session 并重试一次。

## 升级建议

- 使用 139 云盘访问文件、预览 PDF/图片/音视频的用户建议升级。
- 使用 115 网盘云下载并需要选择保存目录的用户建议升级。
- 使用 BoxPlayer 账号/Pro 状态同步的用户建议升级，账号配置和 token 刷新链路更稳定。

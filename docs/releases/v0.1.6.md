# Mnemo v0.1.6

**正式版（stable）**。在 `v0.1.5` 之上修复启动闪退、安装路径与图标/托盘品牌问题，并加固单实例二次启动行为。

## 下载说明

| 平台 | 架构 | 推荐安装包 |
|------|------|------------|
| Windows | x64 / arm64 | `.exe`（NSIS 安装程序） |
| macOS | Intel (x64) / Apple Silicon (arm64) | `.dmg` 或 `.zip` |
| Linux | x64 / arm64 | `.AppImage` / `.deb` / `.pacman` |

- 产品名：**Mnemo**
- 可执行文件：Windows `Mnemo.exe` · macOS `Mnemo.app` · Linux `mnemo`
- 应用 ID：`com.mnemo.app`
- 版本号：`0.1.6`（正式版）

## 本版重点

### 启动与单实例

- 单实例锁提前到 `start()` 之前：避免第二个进程短暂初始化再退出（「打开就闪退」）
- **重复打开快捷方式 / 安装包**：唤起已有主窗口并置前（含隐藏到托盘的情况）
- 未知异常：只提示、不强制退出；渲染进程崩溃可选手动重启

### 安装路径

- NSIS 默认安装目录强制为：
  - 所有用户：`C:\Program Files\Mnemo`
  - 当前用户：`%LocalAppData%\Programs\Mnemo`
- 降低旧注册表 `InstallLocation` 残留导致默认路径怪异的概率

### 图标与托盘

- 窗口 / 托盘使用 `NativeImage` 加载 Mnemo 图标（含内嵌 32px 兜底）
- Windows 托盘与应用图标统一品牌资源（`icon_256x256` / `tray_*`）
- 安装包 `build/` 标准图标资源与 NSIS 安装器图标

### 其它

- Aria 本地引擎 RPC secret 与端口一并回传（随 0.1.5 线保留）

## 安装与升级

1. 建议先退出正在运行的 Mnemo。
2. 安装 `v0.1.6` 对应平台安装包。
3. 若任务栏 / 托盘仍显示旧图标，可重启资源管理器或清理 Windows 图标缓存后再试。

## 质量验证

- 相关单元测试通过（品牌 / 布局断言）。
- 类型检查由 CI 全量执行。
- 正式版由 GitHub Actions 多平台流水线构建并上传到本 Release。

## 版本关系

- `v0.1.5`：正式版（品牌打包加固）
- **`v0.1.6`**：正式版；启动闪退、安装路径、托盘/窗口图标与二次启动唤起
